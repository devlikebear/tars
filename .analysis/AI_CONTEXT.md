## Module Summary

- `auth-plugin-loading`: Enforces API auth policy, loopback console/dashboard exceptions, plugin manifest validation, and trusted plugin skill/MCP boundaries.
- `automation-gateway-browser`: Runs background agent jobs, channel and report surfaces, schedule/ops automation, and browser orchestration outside the chat request lifecycle.
- `client-tui`: Runs one-shot CLI chat and local runtime commands, and provides reusable HTTP/SSE client behavior; the old interactive TUI remains only as a deprecated compatibility stub.
- `configuration-doctor`: Loads, normalizes, and validates runtime configuration with field-table driven YAML/env merge and starter-environment diagnostics.
- `entrypoints`: CLI entrypoints and console-first command surface that choose the runtime path for browser launch, one-shot chat, server, project/ops commands, diagnostics, and hub management.
- `extensions-tools`: Builds unified skill/plugin/MCP snapshots, runs built-in plugin lifecycle hooks, mirrors runtime skills into the workspace, and exposes tool and HTTP surfaces to the server and agent loop.
- `external-integrations`: Bridges TARS to host launchd services, browser relay traffic, human OTP waits, provider credential strategies, and provider-specific transport adapters.
- `project-autopilot-dashboard`: Manages project briefs, tool/workflow policy, board and activity documents, backlog planning, dispatch/review, heartbeat-driven autonomous progress, and dashboard rendering.
- `project-swarm-dogfooding`: Documents how the bundled project workflow skills align with the current autopilot implementation and where gaps remain.
- `semantic-memory`: Indexes explicit memories, compaction summaries, and project documents into embeddings and injects relevant recall into prompts and tools.
- `server-chat-pipeline`: Server bootstrap, API and console routing, chat execution, semantic-memory-aware prompt assembly, per-request tool policy, SSE streaming, and provider/config/runtime endpoints.
- `skillhub-distribution`: Searches the remote TARS Hub, installs or updates skills/plugins into the workspace, and records installed registry state.
- `workspace-data`: Persists sessions, transcripts, projects, usage, schedules, and shared runtime state as local workspace files.
