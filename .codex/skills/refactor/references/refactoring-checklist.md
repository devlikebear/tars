# Refactoring Checklist

## Before Refactoring
- [ ] 모든 테스트가 통과한다
- [ ] 리팩토링 목적이 명확하다
- [ ] 영향 범위를 파악했다 (최대 5개 파일)
- [ ] git 상태가 clean하거나 현재 작업이 커밋되었다

## During Refactoring
- [ ] 한 번에 하나의 리팩토링 기법만 적용한다
- [ ] 기능 변경이 없다 (동작이 동일하다)
- [ ] public API 시그니처를 변경하지 않는다 (명시적 허용 제외)
- [ ] 새 기능을 추가하지 않는다
- [ ] 대량 포맷팅 변경을 하지 않는다

## After Each Step
- [ ] 테스트가 통과한다 (`make test`)
- [ ] 빌드가 성공한다 (`go build ./...`)
- [ ] 린트 경고가 추가되지 않았다 (`go vet ./...`)
- [ ] diff를 확인하여 의도한 변경만 있다

## Final Check
- [ ] 코드 가독성이 개선되었다
- [ ] 중복이 제거되었다 (해당 시)
- [ ] 복잡도가 감소했다 (해당 시)
- [ ] 성능이 저하되지 않았다
- [ ] 문서/주석이 여전히 유효하다
- [ ] 커밋 메시지가 리팩토링 의도를 설명한다

## Red Flags (즉시 중단)
- [ ] 테스트가 실패한다
- [ ] 기능 동작이 변경되었다
- [ ] API 호환성이 깨졌다
- [ ] 예상하지 못한 파일이 변경되었다
- [ ] 성능이 크게 저하되었다
